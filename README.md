В тестировании используется паттерн "Arrange-Act-Assert" (Подготовка-Действие-Проверка)<br>
Пример для понимания паттерна "Arrange-Act-Assert" (Подготовка-Действие-Проверка).<br>
    def add(a, b):
        """
        Эта функция принимает два числа и возвращает их сумму.
        """
        return a + b
<br>
Arrange (Подготовка): Подготавливаем данные. В даном случае это числа 2 и 3 и ожидаемый результат 5.<br>
Act (Действие): Мы выполняем тестируемый код. Это вызов add(2, 3).<br>
Assert (Проверка): Мы сравниваем фактический результат с ожидаемым. Это вся строка с assert.<br>
<br>
<br>

## Синтаксис assert
assert <выражение><br>
<br>
Python вычисляет <выражение>.<br>
Если результат истинный (True), ничего не происходит, и выполнение кода продолжается.<br>
Если результат ложный (False), assert немедленно останавливает выполнение и<br>
вызывает специальное исключение AssertionError.<br>

<p>
Когда функция получает некорректные данные, она не должна:
<ul>
<li>Молча "падать": Просто завершать работу без объяснения причин.</li>

<li>Возвращать странное значение: Например, None, -1 или пустую строку. Это очень опасно, потому что ошибка не будет замечена сразу, а "всплывет" гораздо позже в другой части программы, и найти ее источник будет крайне сложно.</li>

<li>Делать что-то непредсказуемое.</li>
</ul>
Хорошо написанная, надежная функция должна сообщать о проблеме немедленно. Она должна вызвать исключение.
</p>
